# 2026-02-23 Chassis ATTI 自旋模式错配分析（先方案后改码）

## 1. 目标

- 本次要解决的问题：ATTI（`s0=MID`）无摇杆输入时，底盘持续高速自旋（`wz_set` 长期饱和）。
- 预期可观测结果：ATTI 静止输入时 `wz_set` 回到 0 附近，不再贴 `±6000`；GPS（SelfProtect）行为保持不变。

## 3. 关键改动摘要

- 本次不改业务代码，只完成故障路径定位与修复方案落地记录。
- 明确根因不是 PID 参数，而是模式组合错配：`CHASSIS_VECTOR_FOLLOW_GIMBAL_YAW` 与 `yaw_mode=GIMBAL_MOTOR_ENCONDE` 同时出现。

## 4. 关键决策与理由

- 决策 1（故障路径结论）：  
  依据 USB 帧（FireWater 索引）确认 ATTI 故障链路：
  - `11(ch_mode)=0` -> `CHASSIS_VECTOR_FOLLOW_GIMBAL_YAW`
  - `23(yaw_mode)=2` -> `GIMBAL_MOTOR_ENCONDE`
  - `42(rc_ch2)=0`、`44(rc_s0)=3(MID)` -> 无人工旋转输入
  - `15(rel)=-3057mrad`、`16(rel_set)=0`
  - `14(wz_set)=-6000`（持续饱和）
  这表明底盘跟随环在追 `rel_set=0`，而云台 yaw 在相对角模式下不会提供“对地参考”让误差收敛，导致持续满速输出。

- 决策 2（本轮不继续盲改）：  
  按 `docs/00_AI_HANDOFF_RULES.md` 第 6 节，本轮先停止加补丁，先给出“命中实际路径”的修改方案，再实施。

- 决策 3（下一步推荐方案）：  
  将 `HUST_Act` 从“跟随云台角”改为“非跟随角速度模式”（`NO_FOLLOW_YAW` 语义），并增加兜底防护：
  - 兜底规则：若 `chassis_mode==FOLLOW_GIMBAL_YAW` 且 `gimbal_yaw_mode!=GYRO`，则降级为 `NO_FOLLOW` 的 `wz` 计算，避免再次出现持续饱和。

- 备选方案与放弃理由：
  - 备选 A：保持底盘 FOLLOW，不改模式映射，仅调小 `CHASSIS_FOLLOW_GIMBAL_PID_KP`。  
    放弃理由：只能减轻冲击，不能解决“误差不可收敛”的结构性问题。
  - 备选 B：把 MID 档云台改成 GYRO。  
    放弃理由：会改变 ATTI 的云台操控语义（相对角 -> 绝对角），风险高于底盘侧修复。

- 风险 1：若后续只改参数不改模式耦合关系，ATTI 仍会在特定初始角度下持续输出大 `wz_set`。
