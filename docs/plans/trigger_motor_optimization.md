# 拨轮电机控制优化计划

## 对比分析：当前项目 vs HUST_Infantry_2023

### 架构差异

| 维度 | 当前项目 | HUST_Infantry_2023 |
|------|---------|-------------------|
| PID 环路 | **单环**（速度环） | **双环**（位置环+速度环串级） |
| PID 算法 | DJI 基础 PID（无死区、无变速积分） | 增强 PID（死区、变速积分、不完全微分、梯形积分） |
| 热量管理 | 简单阈值判断（裁判系统直读） | 本地预测+裁判系统校准（双轨制） |
| 射弹计数 | 微动开关硬件检测 | 摩擦轮转速扰动软件检测 |
| 连发控制 | 固定速度（15 rad/s） | 根据等级/冷却缓冲动态调速（3500~4500 RPM） |
| 堵转处理 | 速度<1.0 持续700ms→反转500ms | 位置环自带回退 + 反向旋转标志位 |
| 单发精度 | 角度增量 PI_TEN(0.314rad) + 微动开关确认 | 编码器位置增量 Onegrid(36864 ecd) + 位置阈值确认 |
| 摩擦轮调速 | 固定三档 + 手动微调 | 裁判系统弹速等级自动选择 + 每车标定 |
| PC 自瞄 | 无 | 有（Shoot_Tx2_Cal，根据云台误差自动开火） |

### 当前项目的关键缺陷

1. **单环速度 PID 控制拨轮** — 单发精度差，无法精确控制每发的位置增量
2. **PID 算法原始** — 无死区、无变速积分、无不完全微分，容易积分饱和和高频噪声
3. **热量管理被动** — 只读裁判系统数据，无本地预测，存在延迟导致超热量
4. **连发速度固定** — 不根据等级和冷却缓冲动态调整，低等级容易超热量
5. **堵转处理粗暴** — 纯时间判定，无位置回退机制

### HUST 的核心优势

1. **位置+速度双环串级** — 单发时位置环精确控制角度，连发时直接给速度环
2. **变速积分** — 误差大时减少/停止积分，误差小时正常积分，防止超调
3. **不完全微分** — 对微分项做低通滤波（RC_DF），抑制高频噪声
4. **死区** — 小误差不响应，减少抖动
5. **本地热量预测** — 通过摩擦轮扰动检测射弹，本地累加热量，裁判系统数据到达时校准
6. **动态拨轮速度** — 根据机器人等级和冷却缓冲状态选择不同射速

## 优化计划

### Step 1: 增强 PID 控制器

在现有 `pid.c/pid.h` 基础上新增增强型 PID，支持：
- 死区 (DeadZone)
- 变速积分 (I_L / I_U 区间)
- 不完全微分滤波 (RC_DF)
- 梯形积分（取当前误差和上次误差的平均值）

文件：`components/controller/pid.h`, `components/controller/pid.c`

### Step 2: 拨轮电机改为双环串级控制

- 新增位置环 PID (`trigger_pos_pid`)
- 速度环保留为内环
- 单发模式：位置环给定 → 速度环跟踪
- 连发模式：跳过位置环，直接给速度环
- 堵转时：位置环回退一个 grid

文件：`application/shoot.h`, `application/shoot.c`

### Step 3: 热量管理优化

- 新增本地热量预测：通过摩擦轮转速扰动检测射弹
- 裁判系统数据到达时校准本地热量
- 阈值控制：热量达到 95% 上限时停止射击
- 根据冷却速率动态计算可射击窗口

文件：`application/shoot.h`, `application/shoot.c`

### Step 4: 动态连发速度

- 根据裁判系统的机器人等级选择拨轮速度
- 根据冷却缓冲状态（是否有血包加成）调整射速
- 高频模式保留为手动覆盖

文件：`application/shoot.c`

### Step 5: 验证与调参

- 编译验证无错误
- 通过 MCP 遥测工具在线调参验证
- 记录调参结论

## 风险与注意事项

- 双环串级需要重新调参，位置环 Kp 不宜过大
- 热量预测的射弹检测依赖摩擦轮转速反馈质量
- 变速积分的 I_L/I_U 区间需要根据实际误差范围调整
- 改动涉及控制链路，需遵守 `docs/00_AI_HANDOFF_RULES.md`
