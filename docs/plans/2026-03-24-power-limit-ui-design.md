# 功率预测限功与裁判客户端 UI 设计

日期: 2026-03-24
状态: 已确认，待编写实施计划

## 目标

在当前分支中完成两项工作：

1. 将 `main` 中已经验证过方向的“电机功率预测模型 + 功率限制”核心迁移到本分支。
2. 新增一套裁判系统客户端 UI，用于显示底盘模式、射速挡位、退弹成功提示和准心。

本次明确排除以下范围：

- 不移植超级电容相关代码、CAN 协议、检测链路和功率上限覆盖逻辑。
- 不移植 `main` 中为功率调试增加的 USB 遥测列。
- 不扩展更多比赛 UI 元素，只做本次确认的最小可用集合。

## 约束与前提

### 功率限制范围

- 只移植“功率预测模型 + 限功算法本体”。
- 功率算法仍工作在底盘现有控制链路中，不新增任务，不改变 2ms 底盘控制周期。
- 不接入 `supercap.c/.h`，不新增超电相关依赖。
- 不额外迁移 `main` 里其他功率读取、超电发送、遥测扩展逻辑。

### UI 范围

- UI 使用裁判系统客户端自定义图形协议实现，不使用 USB 遥测、OLED 或其他替代显示。
- UI 只包含：
  - 底盘模式：`FOLLOW` / `SPIN`
  - 射速挡位：`LOW` / `HIGH`
  - 退弹成功提示：绿色提示闪烁约 `0.5~1s`
  - 准心：镜式分划线风格
- 不引入新的 UI 设计器生成链路，先用代码内置布局参数实现首版。

## 方案选择

### 功率限制

候选方案：

1. 直接沿用当前分支旧版“按总电流缩放”的限功方式。
2. 迁移 `main` 的电机功率预测模型，但把超电与 USB 调试列一起带进来。
3. 只迁移电机功率预测模型和限功主算法，去掉超电和 USB 调试扩展。

结论：选择方案 3。

原因：

- 用户明确要求“只移植功率预测模型加功率限制，其他都不移植”。
- 这样能把风险压在 `chassis_power_control` 一侧，避免引入超电依赖和额外通信面。
- 现有代码基础上更容易验证行为差异，回滚也更简单。

### UI

候选方案：

1. 直接把 UI 绘制逻辑混入 `referee.c` 或 `referee_usart_task.c`。
2. 新建独立 UI 模块，封装图元、组帧、刷新和闪烁状态机。
3. 先搭一个大而全的 UI 框架，把后续功率/超电/热量等都预埋进去。

结论：选择方案 2。

原因：

- UI 协议组帧与业务状态应解耦，避免把 `referee.c` 变成杂糅的“协议解析 + UI 发送”模块。
- 本次 UI 项目范围较小，单独模块最容易维护和增量扩展。
- 后续即便继续加状态项，也可以在独立 UI 模块中增量演进。

## 功率限制设计

### 总体思路

保留当前底盘控制主链路：

1. `chassis_feedback_update()`
2. `chassis_set_contorl()`
3. `chassis_control_loop()`
4. `chassis_power_control()`
5. `CAN_cmd_chassis(...)`

只替换 `chassis_power_control()` 内部的限功策略。

### 电机功率预测模型

对 4 个底盘电机分别预测功率：

```c
P_motor = torque_coeff * current * speed_rpm
        + k2 * speed_rpm * speed_rpm
        + a * current * current
        + constant;
```

其中：

- `current` 使用 `motor_speed_pid[i].out`
- `speed_rpm` 使用电机反馈 `speed_rpm`
- 总功率只累计正功率项，负功率项不参与总功率约束

设计意图：

- 相比旧版“总电流缩放”，模型法能够区分“高速小电流”和“低速大电流”对功率的不同影响。
- 当电机存在回馈或减速工况时，不直接把负功率算进总功率预算。

### 限功方式

当预测总功率超过功率上限时：

1. 计算目标缩放比例 `power_scale = limit / total_power`
2. 按比例给每个正功率电机分配目标功率
3. 通过一元二次方程反解对应电流
4. 将解回写到 `motor_speed_pid[i].out`

保留原算法的关键特性：

- 使用电机模型反解，而不是简单统一乘一个电流比例
- 不改动负功率电机
- 二次方程无实根时保留原输出，避免数值异常导致电流跳变

### 功率上限来源

本次不迁移 `main` 中更宽的功率读取扩展，仅保留当前项目已有的裁判功率/缓冲读取接口作为输入背景。

但按用户最终要求，本轮只移植“功率预测模型 + 功率限制”本体，因此：

- 不引入超电分支
- 不新增 supercap online/offline 判断
- 不新增超电控制器上报上限覆盖
- 不新增超电发送命令

实现上将采用一个明确的非超电版本：

- 继续基于当前项目已有的裁判数据接口获取底盘功率/缓冲数据
- 实际限功上限仍按当前项目规则或现有常量配置确定
- 不把 `main` 的其他功率读取扩展一起带入

### 数据结构变更

允许为底盘控制结构补充最小必要字段，用于保存算法运行态，例如：

- 预测总功率
- 实际采用的限功上限

这些字段只服务于算法内部观测和后续 UI/调试扩展，不要求本轮接 USB 遥测。

## UI 设计

### 总体结构

新增独立模块，例如：

- `application/rm_ui.h`
- `application/rm_ui.c`

职责：

- 定义裁判客户端 UI 所需图元与字符串
- 组装 `0x0301` 学生交互数据包
- 管理静态绘制与动态刷新
- 管理退弹成功闪烁时序

### 页面布局

#### 准心

- 风格：镜式分划线
- 主体：一条竖线
- 辅助：几条横向长刻线
- 位置：屏幕高度约 `44%`
- 不放在屏幕几何中心，而是略偏下

设计目标：

- 贴近用户描述的“像瞄准镜的线”
- 保持视野中心附近识别性，同时给下方视觉留出更贴近实战的落点感受

#### 左侧状态

- 显示底盘模式
- 仅两态：
  - `FOLLOW`
  - `SPIN`

#### 右侧状态

- 显示射速挡位
- 仅两态：
  - `LOW`
  - `HIGH`

#### 退弹成功提示

- 使用小型绿色提示图元或短文本
- 非常驻
- 在退弹成功后闪烁约 `0.5~1s`
- 超时后恢复默认隐藏或默认状态

### 状态来源

#### 底盘模式

底盘 UI 只关心两态：

- 跟随
- 小陀螺

实现上从现有底盘控制/快照状态取值，并在 UI 模块中映射为：

- `FOLLOW`
- `SPIN`

如果底盘代码中存在更多中间态，本轮 UI 不显示这些中间态，只做两态折叠显示。

#### 射速挡位

当前项目中的射速挡位只有两挡。

UI 不直接使用速度值，而是显示离散挡位标签：

- `LOW`
- `HIGH`

实现上需要为 `shoot.c` 中当前挡位补一个只读 getter，供 UI 模块获取，不把 UI 模块直接耦合到文件内静态变量。

#### 退弹成功

用户定义的成功条件不是“按下了退弹”，而是：

> 拨轮向后装满一个弹丸的位置

因此 UI 事件必须绑定到 reverse 路径真正完成时触发，而不是请求边沿触发。

本轮将按现有 `shoot.c` 语义定义为：

1. 进入 reverse 路径，产生反向一个 `TRIGGER_ONEGRID` 的目标步进
2. 在 `SHOOT_BULLET` 中到达目标位置，即误差满足到位阈值
3. 且该次步进属于 reverse 路径
4. 触发一次 `reverse_success_pulse`

UI 模块收到该脉冲后：

- 点亮绿色提示
- 在 `0.5~1s` 时间窗内闪烁
- 超时自动恢复默认状态

### 发送与刷新策略

- 不新增任务
- 复用现有裁判系统发送链路
- 静态元素一次性绘制
- 动态元素按“状态变化”或“闪烁相位变化”低频刷新

建议刷新策略：

- 常规状态：约 `5~10Hz`
- 闪烁窗口：约 `10~20Hz`

这样可以避免 UI 发送影响实时控制任务。

### 离线恢复

- `REFEREE_TOE` 离线时停止 UI 发送
- 裁判系统恢复后重发静态层和当前动态层
- 避免客户端残留旧图元或显示过期状态

## 风险点

### 功率限制风险

- 电机模型参数若与当前车实际机械/供电状态偏差较大，可能造成限功偏保守或偏激进
- 二次方程反解在极端速度区间可能更敏感，需要关注输出平滑性
- 若当前项目的“功率上限来源”与 `main` 分支假设不一致，迁移时需要严格约束边界，避免把读取逻辑一起误带入

### UI 风险

- 客户端 UI 协议若组帧字段不对，客户端可能无显示或残留旧图形
- 动态刷新频率过高可能压缩裁判串口带宽
- 退弹成功事件若判定点放错，会出现“请求成功但机构未到位就亮灯”或“到位了却不亮”的误判

## 验收思路

### 功率限制

- 编译通过
- 板端观察限功前后底盘响应差异
- 重点确认：
  - 正常运动时不过度限功
  - 大功率工况下能明显抑制输出
  - 不引入异常电流跳变

### UI

- 准心位置在屏幕高度约 `44%`
- 准心样式为“竖线 + 横向长刻线”的镜式分划线
- 底盘模式在 `FOLLOW/SPIN` 间切换正确
- 射速挡位在 `LOW/HIGH` 间切换正确
- 退弹完成一个反向装弹步进并到位后，绿色提示闪烁约 `0.5~1s`
